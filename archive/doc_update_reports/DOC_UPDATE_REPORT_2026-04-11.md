## New Feature Documented This Week

### `execution/OrderManager` (commit `7260c91`, 2026-04-10)

A new `execution/` package introducing the `OrderManager` class — the retry-aware layer that sits between the signal/risk stack and the raw `BinanceRESTClient`.

**Key capabilities:**
- Full order state machine: `PENDING → SUBMITTED → OPEN / PARTIAL_FILL → FILLED / CANCELLED / REJECTED / EXPIRED / ERROR`
- Exponential backoff with jitter for transient errors (HTTP 5xx, network timeouts)
- Rate-limit awareness: honours `Retry-After` on HTTP 429/418
- Idempotent retries via deterministic `client_order_id` (`CSCOPE_{ts}_{sym}_{side}_{rand4}`)
- Non-retryable fast-fail for bad-param errors (Binance codes `-1111`, `-2010`, `-4061`, etc.)
- `cancel_and_replace()` for chasing limit orders on price movement
- `poll_until_terminal()` for synchronous fill confirmation
- Self-Trade Prevention (STP) mode support — Binance Apr-2026 feature (`"NONE"` | `"EXPIRE_TAKER"` | `"EXPIRE_MAKER"` | `"EXPIRE_BOTH"`)
- `on_fill` / `on_error` async callbacks
- 52 unit tests in `tests/test_order_manager.py`
