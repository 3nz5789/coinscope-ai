## New Features Documented (Added This Week)

Based on comparing the committed codebase (commit `ac2d5c7`, 2026-04-03) against the current working tree:

**Trade Journal & Performance layer** — `GET /journal`, `GET /performance`, `GET /performance/daily` are live endpoints backed by the `storage/trade_journal.py` module. Supports querying up to 90 days of trade history and aggregated Sharpe/win-rate/P&L stats.

**Scale-Up Manager** — `GET /scale` and `POST /scale/check` expose the `ScaleUpManager`, which tracks trading performance against promotion thresholds and manages risk tier progression (conservative → standard → aggressive).

**Walk-Forward Validation** — `GET /validate` runs the `WalkForwardValidator` against historical OHLCV data for any symbol and returns fold-by-fold metrics. Previously listed as a roadmap item; now complete.

**Sentiment endpoint** — `GET /sentiment` returns a composite sentiment score (funding rate + open interest data) for any symbol. Previously only available internally to the scoring pipeline.

**Non-secret config introspection** — `GET /config` returns all safe configuration values (scan pairs, thresholds, risk limits) without exposing secrets.
