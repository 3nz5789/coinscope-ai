# coinscope-engine (Cowork plugin)

MCP plugin that wraps the CoinScopeAI Engine API as 15 first-class tools inside Cowork / Claude Code.

## What it provides

A single bundled, self-contained MCP server (`server/index.cjs`, ~1.2 MB, all dependencies bundled) exposing:

**Read-only (14):**

| Tool | Endpoint | Purpose |
| --- | --- | --- |
| `coinscope_health` | `GET /health` | Liveness probe + version |
| `coinscope_config` | `GET /config` | Non-secret config + canonical risk thresholds |
| `coinscope_signals` | `GET /signals` | Latest cached signals + scan loop state |
| `coinscope_scan_status` | `GET /scan/status` | Background scan loop health |
| `coinscope_regime` | `GET /regime` | Market regime per symbol (v3 ML or HMM fallback) |
| `coinscope_sentiment` | `GET /sentiment` | Sentiment score |
| `coinscope_anomaly` | `GET /anomaly` | Anomaly detection output |
| `coinscope_position_size` | `GET /position-size` | Kelly-fractional position sizing |
| `coinscope_exposure` | `GET /exposure` | Portfolio exposure metrics |
| `coinscope_positions` | `GET /positions` | Open positions with unrealised PnL |
| `coinscope_circuit_breaker` | `GET /circuit-breaker` | Circuit breaker state (read only) |
| `coinscope_journal` | `GET /journal` | Trade journal entries |
| `coinscope_performance` | `GET /performance` | Aggregate performance stats |
| `coinscope_decisions` | `GET /decisions` | Recent decisions log |

**Safe trigger (1):** `coinscope_scan_trigger` (POST /scan) — kicks a scan, places no orders.

## Setup

The plugin needs the engine to be reachable. Default URL is `http://localhost:8001` (set via the `COINSCOPE_ENGINE_URL` env var in `.mcp.json`).

To point at a different engine instance, edit `.mcp.json`:

```json
"env": {
  "COINSCOPE_ENGINE_URL": "https://your-engine-host:8001"
}
```

## Safety posture

- **Read-only by default.** Every tool except `coinscope_scan_trigger` is `readOnlyHint: true`.
- **No financial actions.** Order placement, order cancellation, circuit-breaker trip/reset, autotrade enable/disable/config, account sync, decision unpause, and historical backfill are **deliberately excluded** from this plugin. They live in the underlying engine API but were not wrapped — a write-enabled MCP belongs in a separate plugin with per-call confirmation gates.
- Per the CoinScopeAI master prompt: validation phase active through ~2026-05-31. Never "production-ready."

## Usage examples

Once installed, the tools are available as `coinscope_*` in Cowork chat:

- "Run a scan on BTCUSDT and ETHUSDT" → `coinscope_scan_trigger`
- "What's the regime on SOLUSDT right now?" → `coinscope_regime`
- "Size a long BTC at 65000 with stop 64000 on $10k" → `coinscope_position_size`
- "How did this week's trades close?" → `coinscope_journal` + `coinscope_performance`

## Underlying server

This plugin bundles the compiled output of `coinscope-mcp-server` (TypeScript source at `~/Code/CoinScopeAI/coinscope-mcp-server/`). The bundle is generated by `esbuild` and is fully self-contained — no `npm install` step is required on the user's machine.

To regenerate the bundle after changing source:

```bash
cd ~/Code/CoinScopeAI/coinscope-mcp-server
npm run build
npx esbuild src/index.ts --bundle --platform=node --target=node18 --format=cjs --outfile=bundle/index.cjs --packages=bundle
cp bundle/index.cjs ../coinscope-engine-plugin/server/index.cjs
```
